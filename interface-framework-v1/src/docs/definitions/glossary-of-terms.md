# Glossary of Terms
*(Concise, practical, architecture‑friendly definitions)*

- **Feature** — A user‑visible capability of a system.
- **Function** — A callable unit of behavior in code.
- **Method** — A function bound to an object or class.
- **Module** — A self‑contained unit of code or logic.
- **Component** — A reusable building block in UI or architecture.
- **System** — A coordinated set of components working together.
- **Mechanic** — A rule or behavior that defines gameplay.
- **Logic** — The decision‑making rules that govern behavior.
- **Workflow** — A sequence of steps that accomplish a task.
- **Pipeline** — A structured, ordered flow of processing stages.
- **State** — The current condition or configuration of an entity.
- **Trigger** — A condition that initiates an action.
- **Event** — A signal that something has occurred.
- **Handler** — Code that responds to an event.
- **Interface** — A boundary for interaction between systems.
- **Control** — A UI element that accepts user input.
- **Setting** — A configurable parameter.
- **Config** — A structured set of settings.
- **Asset** — A resource such as a texture, model, or sound.
- **Renderer** — A system that draws visuals to the screen.
- **Session** — A runtime instance of user or system activity.
- **Context** — The environment or scope in which operations occur.
- **Profile** — A saved set of preferences or configurations.
- **Manifest** — A declarative description of resources or modules.
- **Adapter** — A bridge between incompatible systems.
- **Orchestrator** — A controller that coordinates multiple agents or processes.
